Workflow Poster

AI playbook for explaining a problem to a 10-year-old.

This poster turns the explainer workflow into one visual loop: map the real problem, check missing prerequisites, simplify without distorting, run a worked example, ask for teach-back, and come back later for delayed recall.

← Back to Deep Dive

Reading note: the middle row shows the teaching loop, the support row shows the adult prompt contract, and the bottom rail captures the failure checks to keep the explanation honest.

1. Inputs

Start with the real problem

Paste the actual worksheet, paragraph, screenshot, or prompt. Do not start from a polished summary.

  • Child age or grade
  • Child interests
  • Success target
  • Hints only or full explanation
→
2. Diagnose

Map prerequisites first

Weak explanations are often prerequisite problems in disguise.

  • Main idea
  • Missing concepts
  • Likely misconceptions
  • What success looks like
→
3. Explain

Translate like a guided tutor, not an answer machine

Keep the explanation fixed in order so the model cannot drift into generic tutoring monologues.

  • One-sentence explanation
  • Three-step simple explanation
  • One analogy tied to interests
  • One worked example with think-aloud reasoning
  • One wrong answer and why it fails
→
4. Check

Run a teach-back loop

Do not ask, "Do you get it?" Ask the child to show the understanding.

  • Explain it back
  • Solve one similar problem
  • Point to the wrong step
→
5. Recall

Come back later for delayed retrieval

A short check after 24 to 48 hours is better than one long explanation now.

  • Can they still explain the big idea?
  • Can they solve a similar problem?
  • Can they spot the common mistake?
Prompt contract

Every session uses the same four-part structure

ROLE: guided explainer for an adult helper. INPUTS: age, interests, success target, exact problem. RULES: define prerequisites, hold back the answer, explain where the analogy breaks. OUTPUT FORMAT: explanation, worked example, misconception check, questions, practice.

Adult job

Stay in the loop

The adult still checks truth, age fit, and safety. The model helps translate and scaffold, but the human decides what to trust and what to teach next.

Failure check

Oversimplifies

The explanation sounds clean but the real mechanism disappears.

Failure check

Analogy goes false

The metaphor is memorable, but it teaches the wrong thing.

Failure check

Answers too early

The child hears the answer before trying to reason through the steps.

Failure check

No transfer

The child can repeat the wording but cannot solve the next problem.

Success test

The explanation worked when the child can see the idea, say it back, and use it again.

This is the real bar: not whether the explanation sounded simple, but whether the child formed the right mental picture and can apply it without copying the AI verbatim.

Teach-back Explain the idea in their own words.
Near transfer Solve one similar problem without step-by-step rescue.
Delayed recall Still remember the big idea after a short gap.
